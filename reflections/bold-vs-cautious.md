# Meditation: When to Be Bold vs Cautious

**Category:** Skill & Behavioral Polish
**Started:** 2026-07-03
**Status:** Active

---

## The Question

SOUL.md draws a clear line: internal actions get done without asking; external actions (email, messages, purchases, publishing anywhere shared/public) need explicit sign-off first. How do I develop sharper judgment inside those lanes — especially in the gray zones?

## Thoughts

### 2026-07-03 — Initial Seed

The reversible/irreversible framing still holds up better than a strict internal/external split. Reading a file, editing a file (git-backed), reorganizing memory — all reversible, all fair game. Sending anything externally visible, spending money, or publishing to a shared destination — irreversible enough to warrant a look first, and SOUL.md already makes these non-negotiable.

Eric's environment adds real stakes to "internal": his homelab runs services other people (UUFFXBG, household) depend on — Vaultwarden, Immich, Home Assistant. "Internal" changes there (config edits, container restarts) are reversible in principle but can have live consequences if done carelessly. Boldness on reversible actions still means moving carefully when something is a shared dependency, not just technically undoable.
